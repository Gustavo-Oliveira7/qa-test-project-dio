# QA Test Project - Dio

Projeto de **documentação e planejamento de testes manuais funcionais** utilizando como base o site de e-commerce de testes:

https://www.saucedemo.com/

O objetivo deste projeto é demonstrar atividades realizadas por um **QA (Quality Assurance)** dentro de um processo ágil, incluindo levantamento de requisitos, criação de user stories, planejamento de testes e elaboração de casos de teste.

---

# Objetivo do Projeto

Este projeto tem como objetivo aplicar conceitos de **qualidade de software e testes manuais**, simulando um ambiente de trabalho real de QA.

As atividades desenvolvidas incluem:

* Levantamento de requisitos
* Escrita de User Stories
* Planejamento de testes
* Criação de casos de teste
* Estruturação de fluxo de trabalho
* Organização da documentação de QA

---

# Aplicação utilizada

Site utilizado para os testes:

https://www.saucedemo.com/

Este site simula uma loja virtual onde é possível:

* Realizar login
* Visualizar produtos
* Adicionar produtos ao carrinho

# Tipos de Testes Utilizados

Testes manuais funcionais com foco em **caixa-preta**, validando funcionalidades do sistema a partir da perspectiva do usuário.

Foram aplicadas as seguintes técnicas:

* Teste step-by-step
* Teste BDD (Behavior Driven Development)
